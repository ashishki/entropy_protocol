# Archive-Only Evidence Mode Decision

Date: 2026-05-05
Decision ID: D-027
Status: APPROVED

## Decision

Current evidence work is restricted to archive-only datasets. Do not run live
monitoring, do not activate streaming feeds, and do not use live/provider uptime
claims to close the current foundation gate.

For the current research foundation, data-stability evidence may be based on
immutable approved archives when the packet records source manifests, hashes,
calendar continuity, missing symbol-days, and unexplained gaps. This can close
the archive-mode data-stability blocker, but it does not prove live feed
stability.

## Rationale

The immediate project objective is to validate the foundation architecture and
evidence discipline before live operation. The project has no budget for paid
providers, and live elapsed-time monitoring would block architectural learning
without improving the archive research baseline.

Archive-only evidence is sufficient for the current question: whether the
implemented protocol foundation can ingest, hash, label, validate, audit, and
package reproducible research datasets without making OOS or live-capital
claims.

## Boundary

This decision does not authorize:

- live capital;
- live broker integration;
- streaming feed activation;
- live data-provider uptime claims;
- OOS/performance claims;
- automatic phase-gate approval;
- synthetic F-30/F-31 closure.

Live/streaming data-stability remains a future hard gate before any live or
production operation.
