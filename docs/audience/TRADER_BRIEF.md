# Entropy Protocol — Краткий бриф для человека рынка
**Классификация:** Confidential — External Contributor Document
**Файл:** `docs/audience/TRADER_BRIEF.md`
**Аудитория:** trader / discretionary PM / crypto / macro / onchain practitioner
**Версия:** 2.0
**Дата:** 2026-03-16

## Что это такое

Entropy Protocol — это не “торговля по чуйке” и не AI, который сам открывает позиции.

Это система, которая берет рыночные идеи и заставляет их проходить через нормальную проверку:
- формулировка гипотезы
- preregistration
- paper / walk-forward evaluation
- решение по фактам

Если коротко: это research and evaluation framework для торговых идей.

## Зачем это нужно

У большинства рыночных идей одна и та же проблема:
- идея звучит разумно
- на графике выглядит убедительно
- но никто не знает, есть там edge или нет

Этот проект нужен, чтобы:
- не путать красивую историю с рабочим сигналом
- не ломать систему бесконечным перебором идей
- проверять идеи одинаковыми правилами

## Что система сможет делать

По мере развития система должна уметь:
- строить базовую systematic long-only модель
- добавлять overlay по режимам рынка
- тестировать short-side расширения
- отдельно считать trading result и treasury yield
- принимать решение “продолжать / убивать / не допускать” по заранее заданным правилам

## Что она не будет делать

Система не будет:
- принимать discretionary решения вместо человека
- пускать идею в портфель только потому, что она “выглядит сильной”
- смешивать market narrative и доказанный signal
- давать AI право напрямую влиять на торговлю

## Как устроен проект простыми словами

### 1. Есть базовая торговая машина

Она генерирует сигналы, аллоцирует риск и считает результат.

### 2. Есть evaluation engine

Он проверяет, работает ли гипотеза вне sample и без утечки данных.

### 3. Есть research layer

Туда попадают новые идеи, в том числе AI-assisted.

### 4. Есть governance

Он не дает новым идеям хаотично пролезать в evaluation и торговую логику.

## Куда в этой системе попадает идея человека рынка

Не в портфель напрямую.

Идея попадает как hypothesis:
- funding signal
- regime flag
- structure level
- flow / liquidity observation
- exit overlay

Потом она проходит путь:
1. формализация
2. human review
3. Trial Registry
4. evaluation

Без этого идея остается просто research note.

## Что от вас полезно

Наиболее полезный вклад:
- понятные рыночные наблюдения
- четкие уровни и условия
- мысли про режимы рынка
- funding / basis / flow patterns
- hypotheses с внятной причинно-следственной логикой

Пример хорошей формулировки:
"если funding > X в течение N окон, то вероятность Y в следующие Z дней растет"

Пример плохой формулировки:
"рынок выглядит перегретым"

## Почему правила такие жесткие

Потому что без них любая research-система быстро превращается в:
- бесконечный перебор сигналов
- confirmation bias
- удобные объяснения задним числом

Здесь это специально заблокировано через:
- preregistration
- Trial Registry
- hypothesis families
- hypothesis budget
- readiness gate

## Фазы проекта

### Phase 0
Строится инфраструктура оценки. Торговых выводов пока нет.

### Phase 1
Проверяется long-only baseline. Это главный тест: есть ли база вообще.

### Phase 2
Добавляется weekly regime overlay, если baseline выдержал проверку.

### Phase 3
Тестируются equity shorts.

### Phase 4
Опционально рассматриваются crypto perp shorts, если экономика не убивает идею funding drag.

### Phase 5
Добавляется treasury layer, но только после доказанной жизнеспособности trading book.

## Почему это может быть полезно человеку рынка

Если у вас сильное чувство рынка, эта система не заменяет его, а дисциплинирует.

Она позволяет:
- превращать наблюдения в проверяемые hypotheses
- не терять хорошие идеи
- быстро отбрасывать слабые идеи
- отделять реальный signal от narrative
