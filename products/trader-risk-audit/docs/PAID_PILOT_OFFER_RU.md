# Paid Pilot Offer RU

Trader Risk Audit - founder-led manual audit для трейдеров, у которых уже есть
executed trades и written risk rules. Это static paid pilot offer, не SaaS
checkout, не account system, не broker connection и не public landing-page app.

## Deliverables

- One manual audit report за один export period.
- Deterministic rule breach table с source row ids.
- Violation-attributed P&L impact.
- Limitations для missing или unsupported fields.
- Telegram-ready summary copy после operator review.
- Short review call или async review notes.

## Required Inputs

- Real trade export за audit period.
- Written risk rules или prop/funded account rules.
- Timezone, session window, account currency и audit period.
- Confirmation, что файлы не содержат API keys, passwords, seed phrases, broker
  tokens, payment identifiers или лишние personal data.

## Timeline

- Intake review: same day, если files complete.
- Manual audit: 48-72 hours после complete inputs и operator-approved mapping.
- Delivery: report and summary после claim-boundary review.

## Privacy Boundary

Files stay local by default. Pilot intake contract задает allowed files and
metadata. Не отправляйте real names, Telegram handles, emails, broker account
ids, screenshots with personal data, payment identifiers, API keys, passwords
или seed phrases.

## No-Advice Boundary

This audit is not investment advice and does not control live trading. Здесь no
broker control, no broker APIs, no signal parsing, no order blocking, no live
risk prevention, no performance prediction и no guaranteed improvement. Это not
PMF evidence само по себе; PMF and market validation require paid pilot
evidence.

## Pilot Price Placeholder

Pilot price placeholder: `$49-$149` за one manual audit report. Final price
confirmed manually before work starts. Checkout flow здесь не реализован.

## Paid Pilot CTA

Пришлите real trade export и written risk rules, подтвердите mapping questions
и купите one paid pilot manual audit. Цель - проверить, достаточно ли useful
report для repeat, а не собирать список новых features.
