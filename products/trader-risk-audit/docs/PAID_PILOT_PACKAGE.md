# Paid Pilot Package

Status: Phase 25 draft package
Date: 2026-05-15
Audience: operator and warm prospects

Trader Risk Audit is offered as a founder-led manual post-trade audit for one
approved export and one set of written risk rules. This package is a paid
concierge pilot, not production SaaS, checkout, hosted upload, broker control,
order blocking, investment advice, or a performance promise.

## What The Prospect Buys

| Item | Included |
|---|---|
| Reviewed audit report | One manually reviewed deterministic report for one approved audit period. |
| Finding traceability | Rule ids, evaluated values, thresholds, timestamps, severity, and source row ids in the reviewed report. |
| P&L attribution | Violation-attributed P&L summary when the source data supports it. |
| Limitations | Unsupported fields, missing data, source-shape caveats, and rejected checks are visible. |
| Delivery packet | Short claim-safe summary copy after manual review. |
| Reproducibility evidence | Manifest, bundle index, and reproducibility status when generated by the local run plan. |
| Review follow-up | One short call or async review note explaining what the report did and did not prove. |

## Required User Input

The pilot starts only after the operator approves:

- one private or anonymized trade export outside git;
- written risk rules, prop rules, or a structured starter-profile selection;
- audit period, timezone, session window, and account currency;
- permission to process the files locally;
- deletion trigger or retention instruction;
- confirmation that the files do not contain unnecessary identifiers,
  credentials, payment identifiers, private notes, or screenshots.

Use `docs/PRIVATE_PILOT_INTAKE_CHECKLIST.md` before accepting any input.

## Turnaround

| Step | Target |
|---|---|
| Intake/redaction review | Same day after complete input is received. |
| Mapping questions | Same day when source columns or written rules are ambiguous. |
| Manual audit run and review | 48-72 hours after complete, approved inputs. |
| Delivery | Only after `docs/PRIVATE_PILOT_REPORT_REVIEW_CHECKLIST.md` reaches `approved_for_manual_delivery`. |

The timeline pauses if input is incomplete, source fields are unsupported,
policy mapping is ambiguous, privacy review fails, or the report needs fixes.

## Pricing Hypothesis

Initial hypothesis: `$49-$149` for one reviewed manual audit report.

This is a hypothesis for warm-user conversations. Final price is confirmed
manually before work starts. No checkout, payment processor, account system, or
payment identifier storage is implemented in this repository.

## Evidence Shown Before The Ask

Use `docs/INTERNAL_DEMO_PACK_OPEN_SOURCE_AUDITS.md` to show artifact quality:

- positive report with deterministic findings and P&L attribution;
- limitation case where leverage is not inferred from missing data;
- rejection case where missing required fields block report generation.

State the boundary clearly: open-source and synthetic artifacts prove report
workflow quality only. They do not prove paid-pilot demand, willingness to pay,
market demand, PMF, customer validation, trading performance, or production
readiness.

## First-User Ask

Use this ask after showing the demo boundary:

> Buy one manual post-trade audit for one approved export and your written risk
> rules. I will return a reviewed report, claim-safe delivery summary,
> limitation notes, and reproducibility evidence when available. It will not
> connect to live trading, place or block orders, give investment advice, or
> promise improved performance.

## Exclusions

The paid pilot does not include:

- SaaS account creation;
- public uploads or hosted storage;
- checkout or payment processor integration;
- exchange write APIs, live broker control, order placement, order cancellation,
  leverage/margin mutation, withdrawal, or transfer access;
- order blocking, automated risk prevention, signal generation, or trading
  recommendations;
- guaranteed performance, loss prevention, or profitability claims;
- tax, legal, compliance, or investment advice;
- unsupported private notes, screenshots, broker statements, support chats, or
  CRM records outside the approved audit input.

## Delivery Gate

External delivery is allowed only when:

- the private run note records a safe non-sensitive label;
- private artifacts stay outside git;
- the report review checklist status is `approved_for_manual_delivery`;
- P0 and P1 counts are zero;
- claim guard and privacy review pass;
- P2 caveats are fixed or visibly accepted;
- deletion trigger is confirmed.

Current repo-visible private run status is blocked:
`docs/private_pilot_runs/pilot_waiting_for_input_001.md` records that no
operator-approved private input has been supplied and no private report has
been run or reviewed.
