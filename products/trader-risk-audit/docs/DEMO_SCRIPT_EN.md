# Two-Minute Demo Script EN

Purpose: founder-led sales call script for Trader Risk Audit. Use it to show a
clear mini-product path and then ask for real evidence: a real trade export,
written risk rules, and a paid manual pilot.

## 0:00-0:20 - Problem

Say:

"Most traders know the rules they meant to follow, but after a trading day it is
hard to see exactly where discipline broke and what P&L impact those breaks
had. This audit takes executed trades plus written rules and produces a
deterministic report."

Do not say: performance will improve, losses were caused by one rule, or the
tool should decide future trades.

## 0:20-0:40 - Upload

Show Telegram or local CLI intake:

- upload trade export;
- upload or select the written policy;
- receive audit id and status;
- operator reviews ambiguous mapping before delivery.

Say:

"The demo uses a public sample. For your audit I need your real trade export
and written risk rules. The public sample is not market validation and is not
PMF evidence."

## 0:40-0:55 - Selected Profile

Show selected profile: `hard`.

Say:

"Soft, medium, and hard are customizable audit presets. They are not investment
advice, not strategy recommendations, and not optimal risk settings. Trader
custom rules and prop/funded account rules have priority. If you already have
custom rules, we audit those instead of forcing a starter profile."

Profile explanation:

- soft: wider thresholds, fewer discipline flags;
- medium: baseline internal validation profile;
- hard: stricter stress-test profile;
- custom rules: the preferred path for real pilots when the trader has written
  rules.

## 0:55-1:25 - Report Summary

Show the top of `demo/public_sample_001/output/report.md`.

Say:

"The report starts with an executive summary: rules reviewed, violations
recorded, affected P&L, and selected profile. Then it shows the detailed
violation table, P&L attribution, limitations, and next review checklist."

Point to:

- report summary;
- rules reviewed;
- violations recorded;
- affected P&L;
- selected profile.

## 1:25-1:45 - Evidence

Show the violation table.

Say:

"Every violation keeps source row ids, timestamp, evaluated value, threshold,
severity, and P&L impact. That means we can trace each flag back to the input
row instead of giving vague coaching feedback."

Point to:

- source row ids;
- timestamp;
- evaluated value and threshold;
- P&L impact;
- limitations for unsupported or missing data.

## 1:45-2:00 - Next Pilot Ask

Say:

"The next pilot ask is simple: send a real trade export and written risk rules,
approve any mapping questions, and buy one manual audit. I am testing whether
this report is useful enough to pay for, not asking you to discuss more
features."

Close with:

"This audit is not investment advice and does not control live trading. It has
no broker APIs, no signal parsing, no order blocking, no auto-advice, and no
performance promise."

## Claim Boundary

- Public sample is internal/demo evidence only.
- Public sample is not market validation, not PMF evidence, not a qualified
  prospect call, not a paid pilot report, not a repeat commitment, and not a
  referral.
- The audit describes rule violations in executed trades. It does not control
  live trading, issue signals, place orders, block orders, or promise trading
  outcomes.
